# Contributing

## Development flow
- Fork the repo.
- Create a feature branch.
- Keep changes focused.
- Add or update tests where relevant.
- Open a pull request with a clear description.

## Code style
- Keep routes thin.
- Keep engine logic in dedicated modules.
- Prefer typed request models.
- Avoid hardcoded scenario data.

## Pull request expectations
- Explain what changed.
- Explain why it changed.
- Include sample request and response payloads when applicable.
- Keep the README up to date if behavior changes.
